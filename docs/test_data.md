# Тестовые данные для /analyze

Подборки текстов для проверки распознавания и маскировки персональных данных в русских и английских запросах. Все примеры можно отправлять через POST `/analyze` с соответствующим языком.

## Русский текст (все поддерживаемые сущности)
```json
{
  "text": "Иванов Иван Иванович из Москвы (ООО \"Контур\"), паспорт 4012 345678, СНИЛС 123-456-789 01, ИНН 7736050003, ОГРН 1027700132195, ОГРНИП 304500116000157, БИК 044525225, р/с 40702810900000001234, к/с 30101810400000000225, телефон +7 (912) 000-00-00 и +90 531 123 4567, email ivan.ivanov@example.com, карта 4111 1111 1111 1111.",
  "language": "ru"
}
```

Покрываемые сущности: `PERSON`, `PHONE_NUMBER_RU`, `PHONE_NUMBER` (международный), `EMAIL_ADDRESS`, `CREDIT_CARD`, `RU_PASSPORT`, `RU_SNILS`, `RU_INN`, `RU_OGRN`, `RU_OGRNIP`, `RU_BIK`, `RU_RS`, `RU_KS` и организационные/географические сущности из модели NER.

## English text (all supported personal data)
```json
{
  "text": "John Doe from London (Contoso Ltd), passport number 4012 345678, email john.doe@example.com, phones +7 (912) 000-00-00 and +44 20 7946 0958, credit card 4111 1111 1111 1111.",
  "language": "en"
}
```

Покрываемые сущности: `PERSON`, `PHONE_NUMBER` (включая российский формат `+7`), `EMAIL_ADDRESS`, `CREDIT_CARD` и географические/организационные сущности, которые вернёт английская NER-модель. Русский паспорт включён для удобства регрессионной проверки, но тип `RU_PASSPORT` будет обнаружен только при обработке текста с `"language": "ru"`.
